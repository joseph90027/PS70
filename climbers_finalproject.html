[ Climbers ]

"Climbers" is a mobile robot that moves along soft surfaces such as fabric and textiles. It navigates up or down the material and rotates using specialized climbing mechanisms. This type of robot could be used in a variety of applications, including search and rescue operations, an inspection of industrial or commercial facilities, or even as a household assistant for tasks such as laundry or cleaning.







[ Future Research Direction ]

The design of such a robot would likely involve several key components in the future, including:

__Mobility system: This could include wheels, tracks, or other mechanisms for movement along soft surfaces, as well as specialized climbing components for navigating the fabric.
__Sensors: The robot would need sensors to help it navigate and avoid obstacles, such as cameras, lidar, or ultrasonic sensors.
__Control system: This would include the hardware and software to process sensor data and direct the robot's movements.
__Power source: The robot would need a reliable power source, such as a battery, to enable it to operate for extended periods.

[ Project Development ]

The first task was to smoothly control an N20 motor and try to make the motor work. It took me some time because it's unlike a traditional servo motor.

Inspired by car mechanisms and the previous research, I modeled the 3d form on 3d software; I used only two motors and tried to fit two engines on each side.



My robot did not move properly in the first version, which is under my expection loll.
As the video showed, the friction of the pla printed part is not enough to make it rotates.


I decided to add gears on both the upper and downer part and print it out.
It rotates successfully, but it works oddly due to the printing quality and accuracy.


Since that it's working, why not test it out on textile?
Besides the gear problem, I found out that the magnetic force is also not strong to make them clip to each other.


In order to not make it over complicated, I decided to remove the gear and just focus on how to increase the friction.
Also, one crucial reason why removing gears is that it will have to install additional chains to turn directions.
So instead, I bought rubber rings from Amazon to test it out.


And then it worked!!!!!!!!!


First time seeing it. It is actually "climbing"!!


But a new problem occurred, the magnetic force decreased after putting on the rubber,
it can only climb toward the ground.


I tried adding more magnets to the parts to increase the clipping force.


It even made it worse.....now it can't even move......


So what's the problem?
It turned out that because the downer part was not a stick anymore,
it would tilt on one side, causing instability and too much friction.


After long consideration, maybe adding more magnets on the wheels would be a better solution.
I also removed the 3d printing wheel covering to maximize the force.


Now it was actually working!!!!!!!!!!!!! woohoo


The robot worked perfectly in one direction.
But I could not turn it in a different direction.


The solution of it was to add a bearing at the downer part,
to let the wheels have the ability to rotate themselves rather than the whole stick move at the same time.


And Finally, after the six versions, I could control the robot like a car!!!!!

        #define MotFwd  19  // Motor Forward pin
        #define MotRev  21 // Motor Reverse pin
        
        #define MotFwd2  22  // Motor Forward pin
        #define MotRev2  23 // Motor Reverse pin 
             
        int encoderPin1 = 18; //Encoder Output 'A' must connected with intreput pin of arduino.
        int encoderPin2 = 5; //Encoder Otput 'B' must connected with intreput pin of arduino.
        
        volatile int lastEncoded = 0; // Here updated value of encoder store.
        volatile long encoderValue = 0; // Raw encoder value
        
        volatile int lastEncoded2 = 0; // Here updated value of encoder store.
        volatile long encoderValue2 = 0; // Raw encoder value
        int encoderPin3 = 10; //Encoder Output 'A' must connected with intreput pin of arduino.
        int encoderPin4 = 9; //Encoder Otput 'B' must connected with intreput pin of arduino.
        
        
        
        void setup() {
          pinMode(MotFwd, OUTPUT);
          pinMode(MotRev, OUTPUT);
          pinMode(MotFwd2, OUTPUT);
          pinMode(MotRev2, OUTPUT);
          Serial.begin(9600); //initialize serial comunication
        
          pinMode(encoderPin1, INPUT_PULLUP);
          pinMode(encoderPin2, INPUT_PULLUP);
          
          digitalWrite(encoderPin1, HIGH); //turn pullup resistor on
          digitalWrite(encoderPin2, HIGH); //turn pullup resistor on
          
          //call updateEncoder() when any high/low changed seen
          //on interrupt 0 (pin 2), or interrupt 1 (pin 3)
          attachInterrupt(5, updateEncoder, CHANGE);
          attachInterrupt(18, updateEncoder, CHANGE);
        
          pinMode(encoderPin3, INPUT_PULLUP);
          pinMode(encoderPin4, INPUT_PULLUP);
          digitalWrite(encoderPin3, HIGH); //turn pullup resistor on
          digitalWrite(encoderPin4, HIGH); //turn pullup resistor on
          attachInterrupt(9, updateEncoder2, CHANGE);
          attachInterrupt(10, updateEncoder2, CHANGE);
        }
        
        void loop() {
        
            if(Serial.available() > 0){           //if some data is available of in the serial port
              char ledPinState = Serial.read();   //read the value
              if(ledPinState == '2'){             //if statement will be true(1)
                digitalWrite(MotFwd, LOW);
                digitalWrite(MotFwd2, LOW);
                digitalWrite(MotRev, LOW);
                digitalWrite(MotRev2, LOW);   
              }
              
              if(ledPinState == '1'){             //if statement will be true(1)
                digitalWrite(MotFwd, LOW);
                digitalWrite(MotFwd2, HIGH);
                digitalWrite(MotRev, HIGH);
                digitalWrite(MotRev2, LOW);   
              }
              if(ledPinState == '0'){             //if statement will be false(0)
                digitalWrite(MotFwd, HIGH);
                digitalWrite(MotFwd2, LOW);
                digitalWrite(MotRev, LOW);
                digitalWrite(MotRev2, HIGH);
              }
        
              if(ledPinState == '3'){             //if statement will be true(1)
                digitalWrite(MotFwd, LOW);
                digitalWrite(MotFwd2, LOW);
                digitalWrite(MotRev, HIGH);
                digitalWrite(MotRev2, LOW);   
              }
              if(ledPinState == '4'){             //if statement will be false(0)
                digitalWrite(MotFwd, LOW);
                digitalWrite(MotFwd2, HIGH);
                digitalWrite(MotRev, LOW);
                digitalWrite(MotRev2, LOW);
              }
            }         
        }
        
        
        void updateEncoder() {
          int MSB = digitalRead(encoderPin1); //MSB = most significant bit
          int LSB = digitalRead(encoderPin2); //LSB = least significant bit
          int encoded = (MSB << 1) | LSB; //converting the 2 pin value to single number
          int sum  = (lastEncoded << 2) | encoded; //adding it to the previous encoded value
          if (sum == 0b1101 || sum == 0b0100 || sum == 0b0010 || sum == 0b1011) encoderValue --;
          if (sum == 0b1110 || sum == 0b0111 || sum == 0b0001 || sum == 0b1000) encoderValue ++;
          lastEncoded = encoded; //store this value for next time
        }
        
        void updateEncoder2() {
          int MSB = digitalRead(encoderPin3); //MSB = most significant bit
          int LSB = digitalRead(encoderPin4); //LSB = least significant bit
          int encoded = (MSB << 1) | LSB; //converting the 2 pin value to single number
          int sum  = (lastEncoded2 << 2) | encoded; //adding it to the previous encoded value
          if (sum == 0b1101 || sum == 0b0100 || sum == 0b0010 || sum == 0b1011) encoderValue2 --;
          if (sum == 0b1110 || sum == 0b0111 || sum == 0b0001 || sum == 0b1000) encoderValue2 ++;
          lastEncoded2 = encoded; //store this value for next time
        }
